# ⚔️ 阶段十：审计与合规 (Audit & Compliance)
**目标**: 证明你的安全工作是有效的。这是企业（甲方）安全团队最核心的 KPI 之一。

## 📝 学习任务

* [ ] **K8s 基线扫描 (CIS Benchmarks)**
  * **工具**: `kube-bench`。
  * **操作**: 运行扫描，查看你的集群有哪些配置不符合 CIS 标准（比如 API Server 参数配置）。
  * **修复**: 尝试修复其中一个 Fail 项（如修改 `--profiling=false`）。
* [ ] **审计日志 (Audit Logging)**
  * **任务**: 开启 K8s 审计日志，记录所有 "Request Received"。
  * **分析**: 假如有人偷偷删除了一个 Secret，你能从日志里找出是谁、什么时间、在哪个节点做的吗？

## 🏆 里程碑练习：合规体检报告

**任务**: 假设计划上市，审计员要求集群必须符合 CIS Benchmark。
1. 运行 `kube-bench` 生成报告。
2. 挑选 3 个 "Fail" 项进行修复。
3. 再次运行，证明已通过。
