# 🛡️ 阶段六：网络微隔离 (Zero Trust)
**目标**: *默认拒绝*。除非我允许，否则谁都别想访问谁。

> **👶 小白概念站**:
> *   **零信任**: 假设内网已经混入了坏人，所以内网之间也不能随意互通。
> *   **NetworkPolicy**: 相当于集群内部的“防火墙规则”。

## 📝 学习任务

* [ ] **实施“默认拒绝”**
  * **实验**: 创建一个 NetworkPolicy，拒绝 `default` 命名空间下的所有入站流量。
  * 验证: 用于 `curl` 测试，你会发现谁都连不上你的服务了。
* [ ] **配置“白名单”**
  * **实验**: 修改策略，只允许带有 `app=frontend` 标签的 Pod 访问 `app=backend` 的数据库端口。
  * 验证: 在 Hubble UI 中，你会看到只有合法的连线是绿色的，其他的都被拦截（红色）。
* [ ] **Cilium 特性 (L7 过滤)**
  * **进阶**: 限制 Pod 只能访问 `google.com`，访问 `baidu.com` 会被丢包。

## 🏆 里程碑练习：死星防御

**任务**: 为阶段一部署的 "Death Star"应用添加 L7 策略，只允许 API 调用 `POST /v1/request-landing`，拒绝其他所有指令（如 `PUT /exhaust-port`）。

**✅ 通关标准**: 手动尝试攻击死星排气口失败 (`403 Forbidden`)，但正常降落请求成功。
