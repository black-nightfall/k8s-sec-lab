# 🔴 阶段五：准入控制 (Admission Control)
**目标**: 把坏人挡在门外。不合规的 Pod 根本不允许启动。

> **👶 小白概念站**:
> *   **准入控制**: 相当于小区的门禁，不符合规定的（比如没戴工牌/以 Root 身份运行）不让进。

## 📝 学习任务

* [ ] **部署策略引擎 (Kyverno)**
  * 操作: 安装 Kyverno。
* [ ] **立规矩**
  * **策略**: “禁止使用 `latest` 标签的镜像”。
  * 验证: 尝试 `kubectl run test --image=nginx:latest`，观察是否被拒绝启动。

## 🏆 里程碑练习：门禁升级

**任务**: 编写一条 Kyverno 策略，强制要求所有 Pod 必须包含 `owner` 标签，且该标签值均不能为空。

**✅ 通关标准**: 尝试部署一个没有标签的 Pod 失败，并收到明确的拒绝信息 "Missing owner label"。
